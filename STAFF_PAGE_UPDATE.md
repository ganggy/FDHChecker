# การแก้ไขหน้า Staff - การตรวจสอบข้อมูลแบบละเอียด

## ✅ สิ่งที่ได้ทำไป

### 1. **API Endpoints ใหม่**

#### `/api/hosxp/funds` (GET)
ดึงรายการกองทุนทั้งหมดจากฐานข้อมูล HOSxP
```json
[
  { "id": "fund_0", "name": "UCS" },
  { "id": "fund_1", "name": "ทหารผ่านศึก" },
  ...
]
```

#### `/api/hosxp/validate` (POST)
ตรวจสอบความถูกต้องของข้อมูลแบบละเอียด
```json
Request:
{
  "records": [
    {
      "hn": "123456",
      "patientName": "นายทดสอบ",
      "fund": "UCS",
      "price": 100
    }
  ]
}

Response:
[
  {
    "hn": "123456",
    "patientName": "นายทดสอบ",
    "status": "สมบูรณ์",
    "issues": []
  }
]
```

### 2. **หน้า Staff ที่ปรับปรุง**

- ✅ ดึงรายการกองทุนจาก API แทนใช้ Mock Data
- ✅ รองรับกองทุนทั้งหมด (16 กองทุนจริง + กองทุนย่อย)
- ✅ แสดงการตรวจสอบข้อมูลแบบละเอียด

### 3. **คอมโพเนนต์ใหม่ - IssuesPanel**

แสดงปัญหาที่พบบ่อยที่สุด ณ ด้านบนของหน้า
- สรุปปัญหาที่พบในข้อมูลปัจจุบัน
- จัดเรียงตามจำนวนครั้งที่พบ
- แสดงสถานะสีเขียว ถ้าข้อมูลสมบูรณ์ทั้งหมด
- แสดงสถานะสีเหลือง ถ้ามีปัญหา

### 4. **การตรวจสอบข้อมูล**

ตรวจสอบฟิลด์ต่อไปนี้:
- ❌ เลขประจำตัวผู้ป่วย (HN)
- ❌ ชื่อผู้ป่วย
- ❌ กองทุน
- ❌ ราคา
- ❌ วันที่บริการ
- ❌ ประเภทบริการ

## 📊 ข้อมูลกองทุนจากฐานข้อมูล

พบทั้งหมด **16 กองทุน**:

1. UCS
2. ทหารผ่านศึก
3. บัตรทอง อสม. (เฉพาะเจ้าตัว)
4. บัตรทองบุคคลในครอบครัว อสม.
5. บัตรทองบุคคลในครอบครัวทหารผ่านศึก
6. บัตรประกันสังคม รพ.สกลนคร
7. บัตรประกันสังคมนอกเครือข่าย
8. บัตรประกันสุขภาพ (ร่วมจ่าย30 บาท)
9. ผู้พิการ
10. ผู้มีรายได้น้อย
11. ผู้สูงอายุ
... และอื่น ๆ

## 🧪 การทดสอบ

### ทดสอบ API Funds
```bash
curl -X GET http://localhost:3001/api/hosxp/funds
```

### ทดสอบ API Validate
```bash
curl -X POST http://localhost:3001/api/hosxp/validate \
  -H "Content-Type: application/json" \
  -d '{
    "records": [
      {
        "hn": "123456",
        "patientName": "นายทดสอบ",
        "fund": "UCS",
        "serviceDate": "2026-03-15",
        "serviceType": "OPD",
        "price": 100
      }
    ]
  }'
```

## ✨ การทำงาน

1. เมื่อเปิดหน้า Staff ระบบจะดึงรายการกองทุนทั้งหมดจากฐานข้อมูล
2. ผู้ใช้สามารถเลือกกองทุนจากรายการแบบเลื่อนลง
3. ตารางแสดงข้อมูลการเบิกจ่าย พร้อมสถานะการตรวจสอบ
4. IssuesPanel แสดงปัญหาที่พบบ่อยที่สุดอยู่ด้านบน
5. การคลิกแถวจะเปิด Modal รายละเอียดพร้อมปัญหาที่พบ
