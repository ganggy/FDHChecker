คำนำ


ระบบศูนย์กลางข้อมูลด้านการเงิน (Financial Data Hub) ของกระทรวงสาธารณสุข เป็นระบบที่ออกแบบมาเพื่อเชื่อมโยงข้อมูลระหว่างระบบสารสนเทศหน่วยบริการ (Hospital Information Systems) กับกองทุนสุขภาพที่เกี่ยวข้อง โดยมีวัตถุประสงค์เพื่อเพิ่มประสิทธิภาพในการบริหารจัดการข้อมูลด้านการเงิน บริการ และทรัพยากรของกระทรวงฯ ให้เป็นไปอย่างมีประสิทธิภาพ และสอดคล้องกับกฎหมายที่เกี่ยวข้อง
ตามระเบียบสำนักงานปลัดกระทรวงสาธารณสุข เรื่อง ระบบศูนย์กลางข้อมูลด้านการเงิน (Financial Data Hub) กระทรวงสาธารณสุข ที่ประกาศ ณ วันที่ 19 มิถุนายน พ.ศ. 2567 ได้กำหนดหลักเกณฑ์ วิธีการ และขั้นตอนการดำเนินการเกี่ยวกับข้อมูลบริการสาธารณสุขของหน่วยบริการในสังกัดสำนักงานปลัดกระทรวงสาธารณสุขอย่างชัดเจน โดยให้หน่วยงานที่เกี่ยวข้องดำเนินการตามแนวทางปฏิบัติและคู่มือการใช้งานที่ปลัดกระทรวงสาธารณสุขได้กำหนดไว้ เพื่อให้การดำเนินงานเป็นไปด้วยความถูกต้อง รวดเร็ว และโปร่งใส โดยคำนึงถึงหลักการคุ้มครองข้อมูลส่วนบุคคลและความมั่นคงปลอดภัยทางไซเบอร์
คู่มือฉบับนี้จัดทำขึ้นโดยกองเศรษฐกิจสุขภาพและหลักประกันสุขภาพ สำนักงานปลัดกระทรวงสาธารณสุข ตามบทบาทของผู้กำกับ ติดตาม และพัฒนามาตรฐานและคุณภาพของข้อมูลในระบบศูนย์กลางข้อมูลด้านการเงิน (Financial Data Hub) กระทรวงสาธารณสุข โดยมีวัตถุประสงค์เพื่อให้ผู้ใช้งาน ซึ่งเป็นหัวหน้าหน่วยบริการ หรือผู้ซึ่งหัวหน้าหน่วยบริการมอบหมาย สามารถนำข้อมูลเข้าระบบได้อย่างถูกต้อง รวดเร็ว และมีประสิทธิภาพ เพื่อให้ข้อมูลที่บันทึกไว้มีความถูกต้อง ครบถ้วน และเป็นปัจจุบัน ซึ่งจะนำไปสู่ การบริหารจัดการด้านการเงินของกระทรวงสาธารณสุขได้อย่างมีประสิทธิภาพยิ่งขึ้น

ส่วนที่ 1 บทนำ
ความเป็นมาและความสำคัญ
ตามรัฐธรรมนูญแห่งราชอาณาจักรไทย พุทธศักราช 2560 มาตรา 55[1] ที่กำหนดให้รัฐต้องดำเนินการให้ประชาชนได้รับบริการสาธารณสุขที่มีประสิทธิภาพอย่างทั่วถึง ประกอบกับหลักการดำเนินงานด้านระบบสุขภาพดิจิทัลของประเทศไทยที่มุ่งเน้นการพัฒนาระบบสุขภาพดิจิทัลเพื่อสุขภาพที่ดีของประชาชนและบรรลุเป้าหมายการพัฒนาที่ยั่งยืน โดยกระทรวงสาธารณสุข เป็นหน่วยงานหลักด้านสุขภาพดิจิทัล          ที่จะต้องสร้างความไว้วางใจให้กับประชาชนโดยปฏิบัติตามกฎหมายที่เกี่ยวข้อง ได้แก่ มาตรฐานความปลอดภัยมั่นคงไซเบอร์ และมาตรการคุ้มครองข้อมูลส่วนบุคคล อีกทั้ง สนับสนุนการวิจัยพัฒนานวัตกรรมสุขภาพดิจิทัล ใช้ข้อมูลที่เป็นหลักฐานเชิงประจักษ์เพื่อขับเคลื่อนนโยบาย (Data Driven Organization)[2]
ระบบศูนย์กลางข้อมูลด้านการเงิน (Financial Data Hub: FDH) เริ่มพัฒนาขึ้นตามนโยบายกระทรวงสาธารณสุข ประจำปีงบประมาณ พ.ศ. 2566 ที่กำหนดว่า "ข้อมูลสุขภาพต้องเป็นของประชาชนและเพื่อประชาชน" และข้อสั่งการของปลัดกระทรวงสาธารณสุขเมื่อวันที่ 13 มิถุนายน 2566 ให้มีการเชื่อมโยงข้อมูลด้านการเงินของกระทรวงสาธารณสุขไปสู่คลังข้อมูลด้านการเงินกระทรวงสาธารณสุข
ในปีงบประมาณ พ.ศ. 2567 Financial Data Hub มีบทบาทสำคัญในการสนับสนุนการเบิกจ่ายของหน่วยบริการกับกองทุนที่เกี่ยวข้องภายใต้โครงการ 30 บาท รักษาทุกที่ ด้วยบัตรประชาชนใบเดียว โดยทำหน้าที่ส่งต่อข้อมูลระหว่างหน่วยบริการและกองทุน ช่วยลดภาระงานการบันทึกข้อมูลซ้ำซ้อน เพิ่มประสิทธิภาพในการเบิกจ่ายงบประมาณ และช่วยวิเคราะห์ข้อมูลเพื่อวางแผนและกำหนดนโยบายอย่างมีประสิทธิภาพ
ปัจจุบัน Financial Data Hub เป็นระบบดิจิทัลที่เชื่อมโยงกองทุนสุขภาพต่างๆ โดยปฏิบัติตามพระราชบัญญัติคุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562 และพระราชบัญญัติความมั่นคงปลอดภัยไซเบอร์      พ.ศ. 2562 แสดงข้อมูลแบบเรียลไทม์ และวิเคราะห์ข้อมูลในรูปแบบ Business Intelligence ซึ่งในปีงบประมาณ พ.ศ. 2567 ได้เชื่อมโยงข้อมูลของหน่วยบริการทั่วประเทศมากกว่า 1,900 แห่ง ครอบคลุมผู้ป่วยมากกว่า 29 ล้านรายการ
อย่างไรก็ตาม ยังมีความท้าทายในการพัฒนาด้านอื่นๆ เช่น การเชื่อมโยงข้อมูลการเบิกจ่ายกับกองทุนที่เกี่ยวข้องอื่นๆ ทั้งภาครัฐและภาคเอกชน และการพัฒนาคุณภาพและความครบถ้วนของข้อมูลที่แสดงบน Dashboard แบบเรียลไทม์ คู่มือฉบับนี้จึงจัดทำขึ้นเพื่อสนับสนุนการดำเนินงานของระบบศูนย์กลางข้อมูลด้านการเงินให้มีประสิทธิภาพและเกิดประโยชน์สูงสุดในการบริหารจัดการระบบสุขภาพดิจิทัลของประเทศ
วัตถุประสงค์ 
1. เพื่อลดภาระงานการบันทึกข้อมูลซ้ำซ้อน
2. เพื่อเพิ่มคุณภาพและประสิทธิภาพการเบิกจ่ายงบประมาณ
3. เพื่อปฏิบัติตามพระราชบัญญัติคุ้มครองข้อมูลส่วนบุคคล พ.ศ.2562 และพระราชบัญญัติความมั่นคงปลอดภัยไซเบอร์ พ.ศ. 2562
4. เพื่อนำข้อมูลมาจัดทำ Data Analytics เพื่อวางแผนและกำหนดนโยบายให้ชัดเจนตรงตามเป้าหมายและมีประสิทธิภาพ


ขั้นตอนและกระบวนการการทำงานของระบบ Financial Data Hub
การส่งข้อมูลจากระบบ HIS เข้าสู่ระบบศูนย์กลางข้อมูลด้านการเงิน (Financial Data Hub) นั้น สามารถทำได้ 2 วิธี คือ
วิธีที่ 1 ส่งข้อมูลผ่านทาง API: เป็นวิธีที่สะดวกและรวดเร็ว โดยระบบ HIS จะเชื่อมต่อกับระบบ Financial Data Hub โดยตรงผ่านทางโปรแกรมที่เรียกว่า API
วิธีที่ 2 ส่งข้อมูลผ่านทาง Web Portal : สำหรับหน่วยบริการที่ไม่สามารถส่งข้อมูลผ่าน API ได้ ระบบได้เตรียมเว็บไซต์สำหรับอัปโหลดข้อมูล โดยหน่วยบริการสามารถเตรียมข้อมูลในรูปแบบที่กำหนดไว้ (Financial Data Set) แล้วอัปโหลดเข้าสู่ระบบได้เลย
ไม่ว่าจะใช้วิธีใด ข้อมูลที่ส่งเข้าระบบ Financial Data Hub ต้องเป็นไปตามรูปแบบที่กำหนดไว้ เพื่อให้กองทุนต่างๆ สามารถนำไปใช้ในการเบิกจ่ายได้อย่างถูกต้องและรวดเร็ว ข้อมูลที่ส่งเข้ามานี้ เรียกว่า Financial Data Set เมื่อหน่วยบริการส่งข้อมูลเข้ามาในระบบแล้ว ระบบจะทำการส่งต่อข้อมูลชุดนี้ไปยังกองทุนที่เกี่ยวข้องโดยอัตโนมัติ ทำให้มั่นใจได้ว่าข้อมูลที่หน่วยบริการส่งจะเป็นข้อมูลเดียวกับที่กองทุนได้รับ และจะไม่มีการแก้ไขข้อมูลระหว่างทาง การใช้ระบบนี้จะช่วยลดขั้นตอนการทำงานที่ซ้ำซ้อน และทำให้การเบิกจ่ายค่าบริการเป็นไปอย่างสะดวก รวดเร็ว และโปร่งใสมากยิ่งขึ้น รายละเอียดดังภาพที่ 1


  

ภาพที่ 1 Financial Data Hub Framework


กล่อง A คือ ขั้นตอนการประมวลผลข้อมูลส่วนบุคคลของผู้ป่วยภายในหน่วยบริการ
เมื่อผู้ป่วย ซึ่งเรียกว่า Data Subject มอบข้อมูลส่วนบุคคลให้กับหน่วยบริการ หน่วยบริการจะได้รับมอบหมายให้ดำเนินการกับข้อมูลเหล่านั้นตามกฎหมายคุ้มครองข้อมูลส่วนบุคคล (PDPA) ซึ่งรวมถึงการทำข้อตกลงการไม่เปิดเผยข้อมูล (NDA) และข้อตกลงการประมวลผลข้อมูล (DPA) กับผู้ป่วย เพื่อที่จะนำมาใช้ในหน่วยบริการ นอกจากนี้ หน่วยบริการยังสามารถส่งข้อมูลที่ได้จากการประมวลผลกลับไปยังระบบศูนย์กลางข้อมูลด้านการเงิน (Financial Data Hub) หรือว่าสำนักงานปลัดกระทรวงในฐานะนิติบุคคล ตามภารกิจ 




กล่อง B คือ ศูนย์กลางข้อมูลของกระทรวงสาธารณสุข
ข้อมูลที่ส่งเข้ามาจะถูกรวบรวมมายัง ศูนย์กลางข้อมูลด้านการเงิน (Financial Data Hub) ของกระทรวงสาธารณสุข เพื่อให้ได้ข้อมูลเชิงลึก (Business Intelligence) โดยข้อมูลจะถูกนำไปสร้างเป็นตารางแสดงข้อมูล (Dashboard) ที่แสดงผลการดำเนินงานของกระทรวงฯ ในภาพรวม แบบ Real-time ช่วยให้ผู้บริหารสามารถติดตาม วิเคราะห์และประมวลผล สู่การนำไปใช้ในการตัดสินใจทางธุรกิจและวางแผนกลยุทธ์ในการบริหารจัดการระบบสาธารณสุขของประเทศ ได้อย่างมีประสิทธิภาพมากขึ้น ทั้งนี้ เพื่อให้การทำงานของระบบเป็นไปอย่างถูกต้องและมีประสิทธิภาพ กระทรวงฯ จะทำข้อตกลงการประมวลผลข้อมูล (Data Processing Agreement) กับผู้ให้บริการ (Vendor) ที่รับผิดชอบในการดูแลระบบ และส่งต่อข้อมูลที่ได้ไปยังหน่วยงานที่เกี่ยวข้อง (กล่อง C) เพื่อนำไปใช้ประโยชน์ต่อไป


กล่อง C คือ ขั้นตอนการแบ่งปันข้อมูลส่วนบุคคลกับหน่วยงานภายนอก ทั้งภายในและภายนอกกระทรวงสาธารณสุข รวมถึงนิติบุคคลอื่นๆ ที่เกี่ยวข้อง (รวมถึงนิติบุคคลที่มีอำนาจเป็น Data Controller ด้วยกัน)
โดยก่อนการแบ่งปันข้อมูล จะต้องมีการทำข้อตกลงการแบ่งปันข้อมูลส่วนบุคคล (Data Sharing Agreement) เพื่อให้แน่ใจว่าการแบ่งปันข้อมูลเป็นไปตามกฎหมายคุ้มครองข้อมูลส่วนบุคคล (PDPA) และเพื่อคุ้มครองสิทธิของผู้ให้ข้อมูล
Data Governance Framework หรือ กรอบการกำกับดูแลข้อมูล คือ ระบบที่ออกแบบมาเพื่อกำหนดแนวทางในการบริหารจัดการข้อมูลภายในองค์กรให้เป็นไปอย่างมีระบบ มีประสิทธิภาพ และมีความปลอดภัย โดยครอบคลุมทุกขั้นตอน ตั้งแต่การรวบรวม จัดเก็บ การวิเคราะห์ ไปจนถึงการนำข้อมูลไปใช้ประโยชน์ โดยขั้นตอนและกระบวนการการทำงานของระบบ Financial Data Hub ดำเนินการตามกรอบ  การดำเนินงานภายใต้ Data Governance Framework ดังภาพที่ 2

ภาพที่ 2 กรอบการดำเนินงานภายใต้ Data Governance Framework


กระบวนการสำหรับการใช้งานระบบ Financial Data Hub (FDH) ของกระทรวงสาธารณสุข (MOPH) ซึ่งแบ่งออกเป็น 3 ขั้นตอนหลัก ดังนี้
1. ลงทะเบียน : หน่วยบริการสามารถสมัครใช้งานระบบ FDH ผ่านแบบฟอร์ม เช่น e-KYC หรือ Google Form โดยดำเนินการตามขั้นตอนที่ระบุไว้ และรอการแจ้งข้อมูลบัญชี (Account) ผ่านทางอีเมล      ที่ลงทะเบียนไว้ หรือผ่านแอปพลิเคชัน/ไลน์ที่เกี่ยวข้อง นอกจากนี้ ยังสามารถเข้าใช้งาน ผ่านระบบ Health Provider ID
2. การเข้าใช้งาน : หน่วยบริการบันทึกผลการใช้บริการ ในระบบ HIS ของหน่วยบริการ หลังจากนั้น ส่งข้อมูลมายังระบบ FDH ด้วย 2 วิธี คือ API หรือ ใช้งานผ่าน Web Portal
3. การส่งข้อมูลและวิเคราะห์ : ข้อมูลเบิกจ่ายค่ารักษาพยาบาลจะถูกส่งเข้าสู่ระบบ Financial Data Hub และระบบจะส่งไปยังระบบของกองทุนสุขภาพที่เกี่ยวข้อง แบบอัตโนมัติ ทั้งนี้ หน่วยบริการสามารถตรวจสอบสถานะหรือวิเคราะห์ข้อมูลผ่าน Business Intelligence (BI) ซึ่งนำเสนอข้อมูลในรูปแบบกราฟหรือรายงาน รายละเอียดดังภาพที่ 3 


  

ภาพที่ 3 กระบวนการสำหรับการใช้งานระบบ Financial Data Hub (FDH) กระทรวงสาธารณสุข


ขั้นตอนและกระบวนการการทำงานของระบบ Financial Data Hub ในหน่วยบริการ สามารถแบ่งออกเป็น 3 กระบวนการ ดังนี้
1. การลงทะเบียน เริ่มจากประชาชนมารับบริการ ยื่นหลักฐานยืนยันตัวตน เวชระเบียนตรวจสอบสิทธิการรักษาพยาบาล
2. การเข้ารับบริการ พยาบาลคัดกรอง ซักประวัติ Vital Sign แพทย์ตรวจรักษา วินิจฉัย และสั่งใช้ยา บันทึกผลการตรวจการรักษา บน HIS เจ้าหน้าที่เคลม บันทึกผลการบริการ (Financial Data Set) ให้มีประสิทธิภาพ ตามที่กองทุนต่าง ๆ ได้วางเกณฑ์ไว้ แล้วกดส่งเข้ามาในระบบ โดยระบบจะรองรับเป็น Real Time แต่จะมี Lacking Time ให้หน่วยบริการไปปรับปรุงข้อมูล 
3. Pre-Audit จะมีการกรองฟิลด์ข้อมูลที่ส่งไปในกองทุนครบหรือไม่ครบ หากไม่ครบจะส่งให้หน่วยบริการปรับปรุงข้อมูล แล้วส่งกลับมา เพราะฉะนั้นข้อมูลที่เข้ามาใน Financial Data Hub จะเป็นข้อมูลที่ค่อนข้างครบและถูกต้อง รายละเอียดดังภาพที่ 4
  

ภาพที่ 4 ขั้นตอนและกระบวนการการทำงานของระบบ Financial Data Hub ในหน่วยบริการ




















































ส่วนที่ 2 วิธีการสมัครขอเข้าใช้งาน


การสมัครขอเข้าใช้งานศูนย์กลางข้อมูลด้านการเงิน Financial Data Hub  กระทรวงสาธารณสุข
การสมัครขอเข้าใช้งานศูนย์กลางข้อมูลด้านการเงิน Financial Data Hub (FDH) กระทรวงสาธารณสุข สามารถดำเนินการได้ 2 ช่องทางคือ 1) ช่องทาง Google Form และ 2) ช่องทาง Account Center (สำหรับผู้ที่เคยใช้งานระบบพิสูจน์ตัวตน) ทั้งนี้ สามารถเข้าเว็บไซต์ กองเศรษฐกิจสุขภาพและหลักประกันสุขภาพที่ https://dhes.moph.go.th/?p=18081 เพื่อดาวน์โหลดแบบคำขอใช้บริการศูนย์กลางข้อมูลด้านการเงิน (Financial Data Hub: FDH) ของกระทรวงสาธารณสุข


  

ภาพที่ 5 ขั้นตอนการขอเข้าใช้งานระบบศูนย์กลางข้อมูลด้านการเงิน (Financial Data Hub) 
กระทรวงสาธารณสุข
1. ดาวน์โหลดแบบคำขอใช้บริการ
ท่านสามารถดาวน์โหลดแบบคำขอใช้บริการศูนย์กลางข้อมูลด้านการเงิน Financial Data Hub (FDH) กระทรวงสาธารณสุขได้ที่ ลิงก์ดาวน์โหลดเอกสาร https://moph.cc/oiUIuvJ2k
2. กรอกข้อมูลและลงนาม
ดำเนินการตามเงื่อนไขการขอใช้บริการ โดยกรอกข้อมูลให้ครบถ้วน และให้ผู้บริหารสูงสุดของหน่วยงานลงนามในช่องผู้ขอใช้บริการให้เรียบร้อย
3. ขอรับบัญชีผู้ใช้งาน (Account)
ท่านสามารถขอรับบัญชีผู้ใช้งานได้ผ่าน 2 ช่องทาง โดยแนบแบบคำขอใช้บริการที่กรอกข้อมูลครบถ้วนแล้ว (ไม่ต้องแนบสำเนาบัตรประชาชน) ทั้งนี้ สามารถขอได้ 1 คนต่อ 1 User และหลายคนต่อ 1 โรงพยาบาล โดยพิจารณาตามภารกิจที่เกี่ยวข้อง
3.1 ช่องทาง Google Form การสมัครผ่าน Google Form เหมาะสำหรับกรณีที่ไม่สามารถสมัครผ่านระบบ MOPH FDH Account Center ได้ โดยเข้าลิงก์สู่การสมัครผ่านช่องทาง Google Form ที่ https://moph.cc/sMC7w9BbF
* กรอกข้อมูลของทุกคนที่มีรายชื่ออยู่ในแบบคำขอผ่านทาง Google Form
* รอการตอบกลับทางอีเมลผู้สมัครภายใน 3 วันทำการนับตั้งแต่วันที่ส่งข้อมูล
ผ่าน Google Form
3.2 ช่องทาง Account Center (สำหรับผู้ที่เคยใช้งานระบบพิสูจน์ตัวตน)
   * สำหรับท่านที่เคยใช้งานระบบพิสูจน์ตัวตน (MOPH eKYC) สามารถสมัครผ่านทาง Account Center
   * กดปุ่ม "สมัครใช้งาน" ที่มุมบนขวาของหน้าเว็บไซต์
  

ภาพที่ 6 สมัครผ่านทาง Account Center


สิ่งที่ต้องเตรียมก่อนสมัครเข้าใช้งาน:
   1. ข้อมูลการลงทะเบียนด้วย eKYC จากระบบพิสูจน์และยืนยันตัวตนสำหรับหน่วยบริการ (หมอพร้อม DID) เพื่อนำ OTP Code มาใช้ในการสมัครบริการ 
   * ดูคู่มือการใช้งานได้ที่ https://moph.cc/QJUQdjsuw
   2. ไฟล์ PDF แบบคำขอใช้บริการศูนย์กลางข้อมูลด้านการเงิน Financial Data Hub (FDH) กระทรวงสาธารณสุข ที่มีการลงนามรับรองจากผู้บริหารระดับสูงของหน่วยงาน 
   * ไฟล์แบบคำขอต้องเป็นสกุล .PDF เท่านั้น ขนาดไม่เกิน 10 MB
   * Download แบบฟอร์มได้ที่ https://moph.cc/pyMPY8suX
ขั้นตอนการสมัครเข้าใช้งาน ผ่านช่องทาง Account Center


ขั้นตอนที่ 1: การยืนยันตัวตน
   1. ระบบจะแสดงหน้าจอให้กรอกข้อมูล
   2. กรอกเลขบัตรประชาชน 13 หลัก (หมายเลข 3)
   3. กรอก Token Code ที่ได้รับจากอุปกรณ์ MOPH-eKYC (หมายเลข 4)
   4. กดปุ่มยืนยัน (หมายเลข 5)
หมายเหตุ: หากกดปุ่มยืนยันแล้วปรากฎหน้าจอแจ้งเตือน ท่านต้องไปลงทะเบียนเป็นเจ้าหน้าที่ประจำหน่วยบริการที่ https://phr1.moph.go.th/idpadmin ส่วนงานทะเบียนบุคลากรประจำหน่วยบริการก่อน จากนั้นกลับมาทำขั้นตอนนี้ใหม่


  


ภาพที่ 7 ภาพตัวอย่างหน้าที่ได้รับ OTP Code สำหรับการสมัครผ่านทาง Account Center
  

ภาพที่ 8 หน้าจอให้ระบุเลขที่บัตรประชาชน และ Token Code สำหรับการสมัครผ่านทาง Account Center


ขั้นตอนที่ 2: กรอกข้อมูลส่วนบุคคล (ตามหมายเลข 7 8 และ 9)
   1. ระบบจะแสดงหน้าจอข้อมูลบุคคลที่ได้จากขั้นตอนการทำ eKYC
   2. ตรวจสอบข้อมูลส่วนบุคคลที่แสดง
   3. กรอกหมายเลขโทรศัพท์มือถือ (หมายเลข 7)
   4. กรอกอีเมล (หมายเลข 8)
   5. ระบุความรับผิดชอบ เช่น ตำแหน่งงานในปัจจุบัน (หมายเลข 9)


  

ภาพที่ 9 แสดงหน้าจอข้อมูลบุคคลที่ได้จากขั้นตอนการทำ eKYC สำหรับการสมัครผ่านทาง Account Center


ขั้นตอนที่ 3: อัปโหลดเอกสาร
   1. เลือกเมนูสำหรับการอัปโหลดเอกสาร (หมายเลข 10)
   2. กดปุ่ม Upload PDF (หมายเลข 11)
   3. เลือกไฟล์ PDF แบบคำขอใช้บริการศูนย์กลางข้อมูลด้านการเงิน Financial Data Hub (FDH) ที่มี   การลงนามรับรองจากผู้บริหารระดับสูงของหน่วยงาน (ตัวอย่างตามภาคผนวก 1)
   4. กดปุ่มอัปโหลด (หมายเลข 13)
   5. กดปุ่มยืนยัน (หมายเลข 14)


  

ภาพที่ 10 หน้าจอการอัปโหลดเอกสาร สำหรับการสมัครผ่านทาง Account Center

การนำข้อมูล 16 แฟ้ม เข้าระบบ Financial Data Hub

การส่งข้อมูลผ่าน Web Portal ซึ่งเป็นทางเลือกสำหรับหน่วยบริการที่ไม่มีความพร้อมในการเชื่อมต่อ API โดยหน่วยบริการสามารถจัดเตรียมข้อมูลตามรูปแบบ Financial Data Set ที่กำหนดไว้ และทำการอัปโหลดผ่านเว็บไซต์ที่จัดเตรียมไว้ให้ ทั้งสองวิธีนี้ช่วยให้หน่วยบริการทุกขนาดสามารถส่งข้อมูลเข้าสู่ระบบกลางได้ตาม ความพร้อมและศักยภาพทางเทคโนโลยีของแต่ละหน่วยงาน

            3. นำเข้าข้อมูล 16 แฟ้ม
3.1 เลือกเมนู “นำข้อมูลเข้า 16 แฟ้ม” ตามภาพที่ 25 หมายเลข 1 หรือกดปุ่ม “นำเข้าไฟล์” ตามหมายเลข 2 เพื่อเข้าสู่หน้า “นำเข้าข้อมูลไฟล์ 16 แฟ้ม” ตามภาพที่ 26


  

ภาพที่ 26 การเลือกเมนู นำเข้าข้อมูลไฟล์ 16 แฟ้ม


  

ภาพที่ 27 หน้านำเข้าข้อมูลไฟล์ 16 แฟ้ม










3.2 ขั้นตอนการนำเข้าข้อมูลไฟล์ 16 แฟ้ม มีดังนี้


3.2.1 เลือกประเภทไฟล์อัปโหลด 1) DBF หรือ 2) TXT แบบมีหัวคอลัมน์ 3) TXT แบบไม่มีหัวคอลัมน์ ตามภาพที่ 28 หมายเลข 1


  

ภาพที่ 28 ขั้นตอนการนำเข้าข้อมูลไฟล์ 16 แฟ้ม


3.2.2 ดูข้อกำหนดอัปโหลดไฟล์เอกสาร ที่ตารางอัปโหลดไฟล์เอกสาร เพื่อนำเข้าข้อมูลไฟล์ 16 แฟ้ม ตามหมายเลข 2
3.2.3 เลือกอัปโหลดไฟล์ ตามประเภทไฟล์อัปโหลด (DBF หรือ TXT) และตารางอัปโหลดไฟล์เอกสาร ตามหมายเลข 5
- ตั้งชื่อไฟล์ตาม ชื่อแฟ้ม ในตารางอัปโหลดไฟล์เอกสาร
- อัปโหลดไฟล์ได้หลายไฟล์ใน 1 ครั้ง ตามภาพตัวอย่างอัปโหลดได้มากกว่า 1 แฟ้ม


  

ภาพที่ 29 ตัวอย่างอัปโหลดได้มากกว่า 1 แฟ้ม


3.2.4 ตรวจสอบการอัปโหลดไฟล์เอกสารว่าไฟล์เอกสารที่ต้องการอัปโหลด อัปโหลดสำเร็จทั้งหมดในตารางอัปโหลดไฟล์เอกสารตามหมายเลข 5
- ดูตัวอย่างไฟล์ที่ตรวจสอบผ่าน ตามภาพตัวอย่างอัปโหลดเอกสารถูกต้อง
- อัปโหลดไฟล์ถูกต้องจะไม่มีข้อความแจ้งเตือน ตามหมายเลข 4 
- อัปโหลดไฟล์ไม่ถูกต้องจะมีข้อความแจ้งเตือน ตามภาพตัวอย่างแจ้งเตือนอัปโหลดเอกสารไม่ถูกต้อง หมายเลข 1


  

ภาพที่ 30 ตัวอย่างอัปโหลดเอกสารถูกต้อง


  

ภาพที่ 31 ตัวอย่างแจ้งเตือนอัปโหลดเอกสารไม่ถูกต้อง


3.2.5 ดูรายละเอียดไฟล์เอกสารที่อัปโหลดสำเร็จ เพื่อตรวจสอบความถูกต้อง กดปุ่ม  “ดูรายละเอียด” ตามภาพที่ 32 หมายเลข 1 หลังกดปุ่ม “ดูรายละเอียด” จะดาวน์โหลดไฟล์ในรูปแบบ .xlsx  
ภาพที่ 32 ผลการอัปโหลดไฟล์ไม่สำเร็จ
           
3.2.6 บันทึกการอัปโหลดไฟล์ หลังอัปโหลดไฟล์ที่ต้องการครบถ้วน กดปุ่ม “บันทึก” ตามภาพที่ 33 หมายเลข 1
  

ภาพที่ 33 บันทึกการอัปโหลดไฟล์
3.2.7 ยืนยันบันทึกการอัปโหลดไฟล์ หลังกดปุ่ม “บันทึก” จะปรากฏหน้าต่างข้อความ “ต้องการบันทึกข้อมูลใช่ไหม” ตามภาพที่ 34
- ถ้ายืนยันบันทึกข้อมูล กดปุ่ม “ตกลง”
- ถ้าไม่ยืนยันบันทึกข้อมูล กดปุ่ม “ยกเลิก”




  

ภาพที่ 34 ยืนยันบันทึกการอัปโหลดไฟล์
           


3.2.8 ทราบผลการอัปโหลดไฟล์
- ถ้าผลการอัปโหลดไฟล์ไม่สำเร็จ ระบบจะแจ้งเตือน ตามภาพที่ 35  
ภาพที่ 35 ผลการอัปโหลดไฟล์ไม่สำเร็จ
           3.2.9 แก้ไขไฟล์เอกสารตามแจ้งเตือน ในข้อ 3.2.8 และกดปุ่ม “ลบเอกสาร” ตามภาพที่ 36 หมายเลข 1 เพื่ออัปโหลดไฟล์ใหม่ที่ทำการแก้ไข แทนที่ไฟล์เอกสารเดิม และบันทึกการอัปโหลดไฟล์ตามขั้นตอนที่ 3.2.6
- ถ้าต้องการอัปโหลดไฟล์เอกสารใหม่ทั้งหมด ให้กดปุ่ม “ลบเอกสารทั้งหมด” ตามหมายเลข 2
  

ภาพที่ 36 การลบไฟล์เอกสาร
การนำข้อมูล Minimal Data Set เข้าระบบ Financial Data Hub
ตามเงื่อนไขการเบิกจ่ายค่าบริการภายใน 72 ชั่วโมง หลังยืนยันการเข้ารับบริการของผู้มีสิทธิหลักประกันสุขภาพแห่งชาติ หน่วยบริการสามารถใช้การแสดงตนยืนยันสิทธิเมื่อสิ้นสุดการรับบริการ โดยดำเนินการพิสูจน์ตัวตน (Authentication) และส่งข้อมูลเบิกจ่ายภายในวันที่รับบริการตามวิธีที่ 6 ผ่านระบบ Financial Data Hub ซึ่งเป็นการใช้ Minimal Data Set เพื่อประกอบการขอรับค่าใช้จ่ายจากกองทุนหลักประกันสุขภาพแห่งชาติให้เป็นไปอย่างรวดเร็วภายใน 72 ชั่วโมงหลังการยืนยันการเข้ารับบริการ
Minimal Data Set ที่ต้องส่งประกอบด้วย:
            * service_date_time (วันที่รับบริการ)
            * cid (เลขบัตรประจำตัวประชาชน)
            * hcode (รหัสหน่วยงานบริการสุขภาพ 5 หลัก)
            * total_amount (ค่าใช้จ่ายรวม)
            * vn (หมายเลขการเข้ารับบริการที่โรงพยาบาล)
            * invoice_number (เลขใบแจ้งหนี้)






วิธี Authentication และใช้งานระบบ FDH API เบื้องต้น
            1. ขอผู้ใช้งานจาก  FDH ACOUNT CENTER  โดยผู้ใช้งานนั้นต้องมีสิทธิ์  MOPH_CLAIM_API  โดย Account นั้นจะต้องมีการ Activate ก่อน และต้องไม่ถูก Suspends
            2. เมื่อเริ่มต้นใช้งาน API ต้องทำการ Login ผ่าน API จาก FDH Account Center เพื่อขอ  jwt จากระบบมาใช้งานใน API ของระบบ Financial Health Gateway ต่อ โดยจะกำหนดได้ผ่าน header authorization: bearer <jwt token>

ส่วนที่ 4 วิธีการใช้งาน Dashboard 


ระบบ Financial Data Hub ได้ออกแบบ Dashboard ที่หลากหลายเพื่อช่วยในการติดตามและวิเคราะห์ข้อมูลสำคัญด้านการให้บริการทางการแพทย์และการเงิน โดยแบ่งเป็น 6 Dashboard หลักที่มีวัตถุประสงค์แตกต่างกัน
                     1. Dashboard Overview: ภาพรวมข้อมูลทั้งหมด
                     2. Dashboard ผลการส่งเคลม UCS: สถานะการเบิกจ่ายจาก สปสช.
                     3. Dashboard ผลการส่งเคลม STP: สถานะการเบิกจ่ายสำหรับผู้มีปัญหาสถานะและสิทธิ
                     4. Dashboard Total Visit: สถิติการเข้ารับบริการ
                     5. Dashboard NCD: ข้อมูลโรคไม่ติดต่อเรื้อรัง
                     6. Dashboard BIO ID: ข้อมูลการพิสูจน์อัตลักษณ์ตัวตนสำหรับคนต่างด้าวและแรงงานต่างด้าว


                     1. Dashboard Overview
แสดงภาพรวมข้อมูลทั้งหมด การใช้งาน Dashboard Overview มีขั้นตอนดังนี้:
                     1. การนำทางในระบบ
                     * ใช้เมนูด้านซ้ายมือเพื่อ Dashboard Overview (หน้าปัจจุบัน) (ตามภาพที่ 40 หมายเลข 1)
                     2. การเลือกช่วงเวลาข้อมูล (ตามภาที่ 40 หมายเลข 2)
                     * กำหนดวันที่เริ่มต้น (จากภาพคือ 09/01/2022)
                     * กำหนดวันที่สิ้นสุด (จากภาพคือ 26/03/2025)
                     * สามารถคลิกที่ไอคอนปฏิทินเพื่อเลือกวันที่ได้โดยตรง
                     3. การกรองข้อมูล (ตามภาที่ 40 หมายเลข 3)
                     * เลือกเขตสุขภาพจากเมนูดรอปดาวน์
                     * เลือกจังหวัดที่ต้องการ
                     * เลือกสังกัดของหน่วยงาน
                     * เลือกหน่วยงานเฉพาะที่สนใจ
                     * เลือกหน่วยบริการ
                     * เลือกประเภทหน่วยบริการ
                     * กดปุ่ม "ค้นหา" (สีเขียว) เพื่อประมวลผลข้อมูลตามเงื่อนไขที่เลือก
                     4. การดูข้อมูลสรุป 
                     * หลังจากกรองข้อมูลแล้ว ระบบจะแสดงผลจำนวนหน่วยบริการที่ส่งข้อมูล และแสดงสัดส่วนประเภทหน่วยบริการผ่านกราฟวงกลม (ตามภาพที่ 40 หมายเลข 4)
                     * แสดงจำนวนที่มารับบริการทั้งหมด และสัดส่วนรูปแบบการส่งข้อมูล (ตามภาพที่ 40 หมายเลข 5)
                     * แสดงข้อมูลการเบิกจ่ายบริการและมูลค่าการเบิกจ่าย โดยแสดงค่ารักษาพยาบาลรวมทั้งหมด และสัดส่วนการเบิกจ่ายระหว่างผู้ป่วยนอก (OPD) และผู้ป่วยใน (IPD) (ตามภาพที่ 40 หมายเลข 6)


  

ภาพที่ 40 Dashboard Overview-1


                     * หัวข้อ: "ตารางแสดงข้อมูลการเคลม" ซึ่งเป็นคอลัมน์ประกอบด้วย: ลำดับ (NO), รหัสหน่วยบริการ, ชื่อหน่วยบริการ, เขตสุขภาพ, API, Web Portal โดยในแต่ละชุดรายการแสดงข้อมูล 10 รายการแรกจากทั้งหมด  ข้อมูลเรียงลำดับตามปริมาณการใช้งาน API จากมากไปน้อย (ตามภาพที่ 41 หมายเลข 7)
                     * หัวข้อ: "โรคที่เข้ารับบริการ" ซึ่งมีแท็บให้เลือกระหว่าง IPD (ผู้ป่วยใน) และ OPD (ผู้ป่วยนอก) โดยปัจจุบันเลือก IPD และมีปุ่มให้เลือกระหว่าง "รักษา" และ "เคลม" โดยเลือก "รักษา" โดยมีเมนูดรอปดาวน์ให้เลือก "โรคหลัก" ซึ่งแสดงรายการโรคต่างๆ (ตามภาพที่ 41 หมายเลข 8)

                     * จำนวนรายการเข้ารับบริการ OPD

ภาพที่ 42 Dashboard Overview-3
                     2. Dashboard ผลการส่งเคลมกองทุนหลักประกันสุขภาพแห่งชาติ (UCS)
ข้อมูลที่แสดงบนกระดาน Dashboard เป็นข้อมูลที่แสดงผลการส่งข้อมูลและการประมวลผลการเบิกจ่าย ซึ่งเป็นการตอบกลับสถานนะ จากสปสช. ผ่านระบบ API และมีการรีเฟรชทุก 5-8 นาที (คลังข้อมูลที่ใช้แสดงผลแยกส่วนจากระบบรับส่ง FDH) ทั้งนี้ การเข้าถึงข้อมูลมีการกำหนดสิทธิ์ตามระดับผู้ใช้งาน โดย รพ. ดูได้เฉพาะข้อมูลของตนเอง สสจ. ดูได้เฉพาะ รพ. ในจังหวัด เขตดูได้เฉพาะ รพ. ในเขตของตน และส่วนกลางสามารถดูภาพรวมได้ทั้งหมด
Dashboard Financial Data Hub นี้เป็นระบบแสดงข้อมูลทางการเงินที่มีการนำเสนอข้อมูลอย่างเป็นระบบ โดยการใช้งาน Dashboard ผลการส่งเคลมกองทุนหลักประกันสุขภาพแห่งชาติ (UCS) ให้เลือกเมนูแถบซ้ายมือ “กองทุน สปสช.” กด “Dashboard” ดังภาพที่ 43 หมายเลข 1 โดยมีรายละเอียดดังนี้:
ส่วนบนของ Dashboard แสดงสรุปข้อมูลทางการส่งข้อมูลการเบิกจ่ายในภาพรวมทุกสิทธิการรักษาพยาบาล (หมายเลข 2) ประกอบด้วย:
                     * จำนวนหน่วยบริการที่ส่งข้อมูล ทั้งหมด 
                     * จำนวนรายการทั้งหมด
                     * จำนวนเงินเรียกเก็บทั้งหมด
                     * จำนวนรายการที่ได้รับการอนุมัติทั้งหมด
                     * จำนวนรายการที่โอนเงินสำเร็จทั้งหมด
ส่วนกลางมีตัวเลือกสำหรับการกรองข้อมูล (หมายเลข 3)
                     * วันที่เริ่มต้นและวันที่สิ้นสุดสำหรับการแสดงข้อมูล
                     * ตัวเลือกเขตสุขภาพ จังหวัด และอำเภอ
                     * ตัวเลือกหน่วยงาน ประเภทหน่วยบริการ และสังกัด
                     * ตัวเลือกกองทุนสุขภาพ
                     * ตัวเลือกหน่วยบริการ
                     * ตัวเลือกประเภทบริการ เช่น OPD, IPD เป็นต้น
ส่วนล่างของหน้าจอแสดงข้อมูลแยกตามประเภทการยื่นทั้งหมด 5 ประเภท (หมายเลข 4)
                     * จำนวนหน่วยบริการที่ส่งข้อมูล ทั้งหมด 
                     * จำนวนรายการทั้งหมด
                     * จำนวนเงินเรียกเก็บทั้งหมด
                     * จำนวนรายการที่ได้รับการอนุมัติทั้งหมด
                     * จำนวนรายการที่โอนเงินสำเร็จทั้งหมด


  
  
  
  
  
  
  
  
  
  
  
  
  

ภาพที่ 43 ผลการส่งเคลมกองทุนหลักประกันสุขภาพแห่งชาติ (UCS)


ล่างถัดมาแสดงตารางแสดงข้อมูลสถานะ ของรายการที่ อยู่ในกระบวนการพิจารณาของ สปสช. จำแนกตามระยะเวลาที่ สปสช. รับข้อมูลไว้แล้ว แต่ยังไม่พิจารณาเสร็จ โดยแยกตามประเภทบริการเป็น IPD (ผู้ป่วยใน) และ OPD (ผู้ป่วยนอก)


  

ภาพที่ 44 ตารางแสดงข้อมูลตามสถานะ – เคสที่อยู่ในกระบวนการพิจารณาของ สปสช.


ภาพที่ 45 แสดงกระบวนการและสถานะข้อมูลการเบิกจ่าย เริ่มจากข้อมูลที่หน่วยบริการส่งเข้าไปยังระบบ ซึ่งมีการจำแนกสถานะเป็นข้อมูลที่สำนักงานหลักประกันสุขภาพแห่งชาติ (สปสช.) รับข้อมูลแล้ว (หมายเลข 1) และข้อมูลที่โครงสร้างไม่เป็นไปตามมาตรฐานกำหนด
จากข้อมูลที่ สปสช.รับข้อมูล (หมายเลข 1) และ ข้อมูลที่ขออุทธรณ์ (หมายเลข 2) มี              การดำเนินการตรวจสอบ แบ่งออกเป็นรายการที่อยู่ระหว่างการพิจารณา อนุมัติ (หมายเลข 4) ไม่อนุมัติ และส่งช่องทางอื่น รวมถึงมีบางส่วนที่ดำเนินการขอทุเลาและยกเลิกการเคลม
ในส่วนที่ได้รับการอนุมัติ (หมายเลข 4) แล้ว จะเข้าสู่กระบวนการโอนเงิน ซึ่งแบ่งสถานะเป็นอยู่ระหว่างการดำเนินการโอนเงิน และโอนเงินสำเร็จแล้ว


  
ภาพที่ 45 ตารางแสดงข้อมูลตามสถานะ


ภาพที่ 46 แสดงตารางแสดงข้อมูลตามสถานะ – Upload หมายถึง ข้อมูล Visit ที่หน่วยบริการส่งเข้าสู่ระบบ FDH แล้ว แต่ยังอยู่ในขั้นตอนของการตรวจสอบ หรือรอการประมวลผลในระดับต่าง ๆ โดยแต่ละสถานะมีความหมายดังนี้
                     * ไม่ประสงค์เบิก หมายถึง ข้อมูล Visit ที่หน่วยบริการไม่ต้องการยื่นขอรับเงินชดเชยจาก สปสช. สำหรับรายการนี้ เช่น กรณีผู้รับบริการอยู่ในสิทธิ UCS แต่เป็นบริการอยู่นอกชุดสิทธิประโยชน์ที่สามารถเบิกได้ หรือเป็นบริการที่อยู่ในเหมาจ่ายรายหัว เป็นต้น (Unclaimed)
                     * รอส่งข้อมูล หมายถึง ข้อมูลยังอยู่ในระบบของหน่วยบริการ ยังไม่ถูกส่งเข้าสู่ระบบกลางของ สปสช. โดยปกติ สปสช. จะรับข้อมูล ประมาณ 5 นาที หลังส่งข้อมูลเข้าระบบ (Received)
                     * สปสช. รับข้อมูล หมายถึง ข้อมูลได้รับโดย สปสช. แล้ว โดยเตรียมเข้าสู่การตรวจสอบเพื่อพิจารณาเบิกจ่ายต่อไป
                     * โครงสร้างไม่ได้มาตรฐานตามกำหนด หมายถึง ข้อมูลที่ส่งมีโครงสร้างไม่เป็นไปตามรูปแบบที่ สปสช. กำหนด เช่น ฟิลด์ข้อมูลไม่ครบ รูปแบบข้อมูลผิดพลาด หรือไม่เป็นไปตามมาตรฐานที่ระบบกำหนดไว้ (Incompleted)
  

ภาพที่ 46 ตารางแสดงข้อมูลตามสถานะ – Upload


ภาพที่ 47 แสดงตารางแสดงข้อมูลตามสถานะ – รายการอนุมัติ เป็นข้อมูล Visit ที่ สปสช.รับข้อมูล  และอยู่ในกระบวนการพิจารณาของ สปสช. หรือรอการประมวลผลในระดับต่าง ๆ โดยแต่ละสถานะมีความหมายดังนี้
                     * อยู่ในกระบวนการพิจารณาของ สปสช. หมายถึง ข้อมูลที่ส่งเข้าสู่ สปสช. แล้ว และกำลังอยู่ระหว่างกระบวนการตรวจสอบ พิจารณาความถูกต้อง ความครบถ้วน และเงื่อนไขการเบิกจ่าย (Waited)
                     * อนุมัติ หมายถึง รายการที่ผ่านการพิจารณาแล้วโดย สปสช. ว่าเข้าเงื่อนไขการเบิกจ่าย สามารถนำเข้าสู่ขั้นตอนโอนเงินได้ต่อไป (Approved)
                     * ไม่อนุมัติ (คงเหลือ)        หมายถึง รายการที่ผ่านการตรวจสอบแล้ว แต่ ไม่ผ่านเกณฑ์     การอนุมัติ เช่น ข้อมูลไม่ถูกต้อง บริการไม่ตรงสิทธิ หรืออยู่นอกเงื่อนไขการเบิก  (Rejected)
                     * ส่งช่องทางอื่นแล้ว หมายถึง รายการที่ไม่ได้ดำเนินการต่อในระบบนี้ เนื่องจากได้ดำเนินการผ่านช่องทางการเบิกอื่นแล้ว หรือพบว่าข้อมูลซ้ำซ้อน Visit ที่ติด C179 (Duplicated)


  

ภาพที่ 47 ตารางแสดงข้อมูลตามสถานะ – รายการอนุมัติ


ภาพที่ 48 แสดงตารางแสดงข้อมูลตามสถานะ – สถานะโอนเงิน เป็นข้อมูล Visit ที่ผ่าน        การพิจารณาแล้วโดย สปสช. ว่าเข้าเงื่อนไขการเบิกจ่าย สามารถนำเข้าสู่ขั้นตอนโอนเงินได้ต่อไป โดยมีกระบวนในระดับต่าง ๆ โดยแต่ละสถานะมีความหมายดังนี้
                     * รอตัดรอบการเบิกจ่าย หมายถึง รายการที่ได้รับการอนุมัติแล้ว แต่ยัง รอรอบการจ่ายเงินตามรอบบัญชี ซึ่งอาจขึ้นกับรอบงวดของการโอน หรือการจัดสรรงบประมาณ (Approved)
                     * รอโอนเงิน        หมายถึง รายการที่ผ่านการอนุมัติแล้ว และอยู่ในขั้นตอน รอการดำเนินการโอนเงินจาก สปสช. ไปยังหน่วยบริการ (Cut off Batch)
                     * โอนเงินสำเร็จ หมายถึง รายการที่ได้รับการโอนเงินเข้าบัญชีหน่วยบริการเรียบร้อยแล้ว รวมถึงมีการระบุยอด เงินชดเชยพึงรับ ซึ่งเป็นจำนวนเงินที่ควรได้รับตามสิทธิ (Settled)


  

ภาพที่ 48 ตารางแสดงข้อมูลตามสถานะ – สถานะโอน


ภาพที่ 49 แสดงตารางแสดงข้อมูลตามสถานะ – รายการอุทธรณ์ โดยมีรายการสถานะ และคำอธิบายเพิ่มเติม ดังนี้
                     * ขออุทธรณ์        หมายถึง หน่วยบริการไม่เห็นด้วยกับผลการพิจารณาที่ไม่อนุมัติ และได้ดำเนินการ ยื่นอุทธรณ์ต่อ สปสช. เพื่อขอให้พิจารณาใหม่ตามข้อมูลหลักฐานเพิ่มเติม
                     * ยกเลิกการเคลม หมายถึง หน่วยบริการเป็นผู้ดำเนินการ ยกเลิกการขอเบิกค่าบริการเอง เช่น พบว่าเคสไม่เข้าเกณฑ์ หรือกรอกข้อมูลผิดพลาดที่ไม่สามารถแก้ไขได้


  

ภาพที่ 49 ตารางแสดงข้อมูลตามสถานะ - รายการอุทธรณ์




ภาพที่ 50 แสดงการส่งใหม่กรณีรายการที่ไม่อนุมัติ (แก้ไข C) โดยมีรายการสถานะและคำอธิบายเพิ่มเติม ดังนี้
                     * ไม่อนุมัติ (ยอดสะสม) หมายถึง รายการเคลมที่หน่วยบริการส่งเข้ามาแล้ว ไม่ผ่าน     การอนุมัติสะสมรวมทุกงวด
                     * ส่งใหม่แก้ไขรายการที่ไม่อนุมัติ หมายถึง รายการที่เคยไม่ผ่านการอนุมัติในรอบก่อนหน้า และหน่วยบริการได้ ดำเนินการแก้ไขหรือส่งใหม่ เข้าระบบแล้ว เพื่อขอให้ สปสช. พิจารณาใหม่อีกครั้ง
                     * อนุมัติ หมายถึง รายการที่ส่งใหม่แล้วได้รับอนุมัติในการพิจารณารอบใหม่ เรียบร้อยแล้ว โดยอยู่ในกลุ่มที่เคยไม่ผ่านแต่ได้รับการแก้ไขสำเร็จ
                     * อยู่ในกระบวนการพิจารณาของ สปสช. หมายถึง รายการที่ส่งใหม่แล้ว และ กำลังอยู่ระหว่างการตรวจสอบ ของ สปสช. ยังไม่สรุปผลอนุมัติหรือไม่อนุมัติ
                     * ไม่อนุมัติ (คงเหลือ)        หมายถึง รายการที่ผ่านการพิจารณารอบใหม่แล้ว แต่ยังคงไม่ผ่านการอนุมัติ เช่นเดียวกับรอบก่อนหน้า แม้มีการแก้ไขหรือส่งใหม่แล้วก็ตาม


หมายเหตุ : จำนวนส่งใหม่แก้ไขรายการไม่อนุมัติจะไม่เท่ากับผลรวมของรายการอนุมัติ , รายการอยู่ในกระบวนการพิจารณาของสปสช. และรายการไม่อนุมัติคงเหลือ เนื่องจาก 3 รายการ ซึ่งได้แก่ 1) รายการติด C และส่งใหม่เป็นสถานะไม่ประสงค์เบิก 2) รายการติด C และส่งใหม่เป็นสถานะโครงสร้างไม่ได้ตามมาตรฐาน  3) รายการติด C และส่งใหม่เป็นสถานะส่งช่องทางอื่นแล้ว ไม่ได้แสดงที่หน้า Dashboard


  

ภาพที่ 49 ตารางแสดงข้อมูลตามสถานะ - รายการอุทธรณ์


ภาพที่ 51 แสดงการจองเคลม โดยเป็นผลการส่งเลข Authen + Minimal Data Set ของ  หน่วยบริการ เพื่อเป็นการยืนยันสิทธิเมื่อสิ้นสุดการบริการตามนโยบาย 30 บาท รักษาทุกที่ด้วยบัตรประชาชนใบเดียว ซึ่งหน่วยบริการสามารถส่งข้อมูลเพื่อขอรับค่าใช้จ่ายบริการสาธารณสุขได้ภายหลังได้ เปรียบเสมือนการจองเคลม โดยมีรายการสถานะและคำอธิบายเพิ่มเติม
                     * จองเคลม (ยอดสะสม) หมายถึง รายการที่หน่วยบริการได้ดำเนินการ ส่งเลข Authen + Minimal Data Set เพื่อเป็นการยืนยันสิทธิเมื่อสิ้นสุดการบริการ
                     * ส่งเคลมแล้ว หมายถึง รายการที่หน่วยบริการ ดำเนินการส่งข้อมูลการเคลมจริงเข้าระบบเรียบร้อยแล้ว หลังจากที่มีการยืนยันสิทธิเมื่อสิ้นสุดการบริการไว้ก่อนหน้านี้
                     * จองเคลม (คงเหลือ) หมายถึง รายการที่หน่วยบริการได้จองไว้ แต่ยังไม่ได้ส่งเคลมจริงเข้าระบบ
  

ภาพที่ 51 จองเคลม


ภาพที่ 52 แสดงรายการเคลมตามสถานะประเภทบริการ (หน่วย:เคส) 
                     * หมายเลข 1 แสดงข้อมูลผลการเคลมตามสถานะประเภทบริการ (หน่วย : เคส) ปัจจุบันมี 2 บริการที่แสดง คือ IPD และ OPD แต่ละสถานะสอดคล้องกับหมายเลข 8 (ยกเว้น ไม่ประสงค์เบิก จะไม่แสดงในตารางนี้)
                     * หมายเลข 2 ประเภทบริการ OPD จะแยกออกเป็นบริการเฉพาะ หมายถึง Visit ที่มีการโอนเงิน ไม่เท่ากับศูนย์บาท เหมาจ่ายรายหัว หมายถึง Visit ที่มีการโอนเงิน เท่ากับ ศูนย์ บาท
                     * หมายเลข 3 ร้อยละของการชดเชย หมายถึงการนำค่าเงินชดเชยพึงรับ หารด้วยยอดเรียกเก็บ คูณด้วย 100 โดยจำแนกออกเป็นแต่ละประเภทบริการ
  

ภาพที่ 52 รายการเคลมตามสถานะประเภทบริการ (หน่วย:เคส)


ภาพที่ 53 กราฟแสดงข้อมูลหน่วยบริการที่ส่งข้อมูลรายวัน (หน่วย : แห่ง) จะแสดงจำนวนหน่วยบริการที่ส่งข้อมูลเข้า FDH ช่วงระยะเวลา 30 วันนับจากวันที่สิ้นสุดการกรองย้อนหลัง
  

ภาพที่ 53 จำนวนหน่วยบริการที่ส่งข้อมูลรายวัน (แห่ง)


ภาพที่ 54 กราฟแสดงการส่งข้อมูลรายวัน (หน่วย : เคส) จำแนกออกเป็น IPD และ OPD สามารถเลือกกดปุ่มได้ หมายเลข 15.1 แสดงจำนวน Visit Date ช่วงระยะเวลา 30 วัน นับจากวันที่สิ้นสุด  การกรองย้อนหลัง
  

ภาพที่ 54 การส่งข้อมูลรายวัน (เคส)


ภาพที่ 55 แสดงรายการเข้ารับบริการทั้งหมดตามสิทธิ สิทธิหลักประกันสุขภาพ (รายการอนุมัติ) กราฟแท่งแสดงการเข้ารับบริการทั้งหมด เป็นข้อมูล Visit Date แสดงจำนวนตามสถานะต่างๆ           4 สถานะ ได้แก่ รายการทั้งหมด สปสช.รับข้อมูล อนุมัติ และโอนเงินสำเร็จ
ภาพที่ 56 เป็นกราฟแท่งแสดงการเข้ารับบริการทั้งหมด *เลือกดูรายการที่ไม่อนุมัติ (คงเหลือ) โดยจะแสดงจำนวน ตามสถานะต่างๆ 3 สถานะ ได้แก่ รายการทั้งหมด สปสช.รับข้อมูล และไม่อนุมัติ (คงเหลือ)
  

ภาพที่ 55 รายการเข้ารับบริการทั้งหมดตามสิทธิ สิทธิหลักประกันสุขภาพ (รายการอนุมัติ)
  

ภาพที่ 56 รายการเข้ารับบริการทั้งหมดตามสิทธิ สิทธิหลักประกันสุขภาพ (รายการไม่อนุมัติ)


ภาพที่ 57 หน้าจอแสดงสัดส่วนของ 5 อันดับข้อมูลรหัสติด C คงเหลือ และภาพที่ 58 หน้าจอแสดงตารางข้อมูลรหัสติด C จำแนกออกเป็น C คงเหลือ ส่งใหม่แก้ไข C และ C ทั้งหมด


*หมายเหตุ : 1 รายการเคลม/เคส สามารถติด C ได้มากกว่า 1 รหัส (จำนวน C ในตารางอาจจะมากกว่าจำนวนเคสที่ติด C)

                     3. Dashboard ผลการส่งเคลมกองทุนประกันสุขภาพสำหรับบุคคลที่มีปัญหาสถานะและสิทธิ (STP)
แสดงข้อมูลผลการเบิกจ่ายค่ารักษาภายใต้กองทุน STP โดยเลือกกองทุน “STP” จากแถบซ้ายมือ และเลือก “Dashboard” เพื่อดูภาพรวมผู้มีสิทธิ รายการเคลม รายโรคยอดนิยม และการใช้บริการจำแนกตามพื้นที่และหน่วยบริการ 
ภาพที่ 61 แสดงหน้าหลัก Dashboard STP โดยแสดงภาพรวมของหน่วยบริการทั้งหมดที่ให้บริการกลุ่ม STP ทั่วประเทศ มีช่องเลือกกรองข้อมูลตามปีงบประมาณ จังหวัด หน่วยบริการ และสถานะของผู้มีสิทธิ (Active/Inactive) แสดงจำนวนผู้มีสิทธิที่ลงทะเบียน และงบประมาณที่ใช้ไป นอกจากนี้ แสดงข้อมูลผู้มีสิทธิแยกตามเพศ ช่วงอายุ และประเภทบริการ


  

ภาพที่ 61 Dashboard ผลการส่งเคลมกองทุนประกันสุขภาพ
สำหรับบุคคลที่มีปัญหาสถานะและสิทธิ (STP)-1


ภาพที่ 62 แสดงรายละเอียดผลการเคลม โดยแสดงจำนวนรายการที่มีการเบิกเคลมทั้งหมด แยกเป็น IPD และ OPD มีสรุปรายการแยกตามประเภทบริการ พร้อมแสดงจำนวนเคสและสถานะของการเบิก (เช่น อยู่ระหว่างพิจารณา อนุมัติ ฯลฯ) ข้อมูลการเคลมถูกแสดงในรูปแบบตารางเพื่อเปรียบเทียบผลลัพธ์ของแต่ละสถานะได้อย่างละเอียด รวมถึงแสดงข้อมูลของหน่วยบริการที่ยื่นเคลม ทั้งในภาพรวมและจำแนกตามประเภทการให้บริการ


  

ภาพที่ 62 Dashboard ผลการส่งเคลมกองทุนประกันสุขภาพ
สำหรับบุคคลที่มีปัญหาสถานะและสิทธิ (STP)-2


ภาพที่ 63  แสดงกราฟแสดงโรคและโรงพยาบาลยอดนิยม โดยกราฟด้านซ้ายแสดง 10 อันดับโรค ที่มีการเบิกเคลมสูงสุด โดยแสดงเป็นกราฟแท่งตามจำนวนเคส และกราฟด้านขวาแสดง 10 อันดับโรงพยาบาล ที่มีจำนวนผู้มีสิทธิเข้ารับบริการมากที่สุด


  

ภาพที่ 63 Dashboard ผลการส่งเคลมกองทุนประกันสุขภาพ
สำหรับบุคคลที่มีปัญหาสถานะและสิทธิ (STP)-3


ภาพที่ 64  แสดงแผนที่ประเทศไทยและการแสดงผลรายเขตสุขภาพ โดยฝั่งซ้ายแสดงแผนที่ประเทศไทย โดยไฮไลต์พื้นที่ตามจำนวนผู้ใช้บริการหรือจำนวนเคลมในแต่ละจังหวัด ฝั่งขวาแสดงกราฟแท่งเปรียบเทียบข้อมูลราย เขตสุขภาพ สำหรับ IPD และ OPD ด้านล่างสุดมีกราฟแสดงจำนวนการเคลมจำแนกรายโรงพยาบาล พร้อมตัวเลือกกรองตามประเภทบริการ
  

ภาพที่ 64 Dashboard ผลการส่งเคลมกองทุนประกันสุขภาพ
สำหรับบุคคลที่มีปัญหาสถานะและสิทธิ (STP)-4










                     4. Dashboard Total Visit – สถิติการเข้ารับบริการ
แสดงภาพรวมการส่งข้อมูลการเข้ารับบริการของโรงพยาบาลทั่วประเทศ โดยสามารถกรองตามช่วงเวลา เขตสุขภาพ จังหวัด หน่วยงานต้นสังกัด และระดับหน่วยบริการ เพื่อดูอัตราการส่งข้อมูล และเปรียบเทียบจำนวนโรงพยาบาลที่ส่งข้อมูลครบถ้วนหรือยังไม่ส่งข้อมูลในแต่ละเขตสุขภาพ
ภาพที่ 65 Dashboard Total Visit (ภาพบน) แสดงสัดส่วนโรงพยาบาลที่ส่งข้อมูลการเข้ารับบริการ แยกตามเขตสุขภาพ พร้อมทั้งแสดงจำนวนโรงพยาบาลที่ส่งข้อมูลครบและยังไม่ส่งในแต่ละเขตผ่านกราฟแท่ง

แสดงข้อมูลการให้บริการและค่าใช้จ่ายในกลุ่มโรคเรื้อรัง NCDs 6 โรค ได้แก่ โรคเบาหวาน [E10-E14] โรคความดันโลหิตสูง [I10-I15] โรคปอดอุดกั้นเรื้อรัง [J40-J47] โรคมะเร็ง [C00-C96] สุขภาพจิต [F04-F99, G47] และโรคหัวใจและหลอดเลือด [I01-I99, G45-G46.8, B332 โดยสามารถกรองตามช่วงเวลา เขต จังหวัด และหน่วยบริการ

                     * จำนวนหน่วยบริการและผู้ป่วยที่รับบริการ
                     * จำนวนครั้งการรับบริการและค่าใช้จ่ายรวม
                     * การเปรียบเทียบข้อมูลรายโรคในรูปแบบกราฟ
                     * แนวโน้มการใช้บริการและค่าใช้จ่ายรายเดือน เพื่อใช้วางแผนและติดตามภาระค่าใช้จ่ายของระบบสุขภาพอย่างมีประสิทธิภาพ

                     6. Dashboard BIO ID
Dashboard นี้แสดงข้อมูลการดำเนินการลงทะเบียน BIO ID สำหรับบุคคลต่างด้าวหรือกลุ่มประชากรเปราะบางที่ไม่มีเลขประจำตัวประชาชน เพื่อให้สามารถเข้าถึงสิทธิด้านสุขภาพได้อย่างถูกต้อง ข้อมูลประกอบด้วย:
                     * จำนวนผู้ลงทะเบียนทั้งหมด พร้อมกราฟแสดงสัดส่วนการดำเนินงานในแต่ละเขตสุขภาพ
                     * รายละเอียดผลการลงทะเบียน เช่น จำนวนผู้ลงทะเบียนรายวัน รายเขต และหน่วยงานที่เกี่ยวข้อง
                     * การจำแนกผู้ลงทะเบียนตามเพศ ช่วงอายุ และสถานะการถือสิทธิ


ภาพที่ 70 แสดงภาพรวมการลงทะเบียน BIO ID โดยแสดงจำนวนผู้ที่ได้รับการลงทะเบียน BIO ID ทั้งหมด พร้อมกราฟแสดงสัดส่วนการลงทะเบียนในแต่ละเขตสุขภาพ แยกเป็นจังหวัดที่ดำเนินการแล้วและยังไม่ดำเนินการ


  

ภาพที่ 70 สถิติการลงทะเบียน Bio-ID-1


ภาพที่ 71 รายละเอียดผลการลงทะเบียน BIO ID แสดงจำนวนผู้ลงทะเบียนแยกรายวัน    รายเขตสุขภาพ และแหล่งข้อมูล รวมถึงหน่วยงานที่ดำเนินการลงทะเบียน และสัญชาติของผู้ถือ BIO ID 
  

ภาพที่ 71 สถิติการลงทะเบียน Bio-ID-2


ภาพที่ 72 แสดงข้อมูลเพศและอายุของผู้ลงทะเบียน BIO ID แสดงสัดส่วนผู้ลงทะเบียน    BIO ID แยกตามเพศและช่วงอายุ เพื่อสะท้อนโครงสร้างประชากรในกลุ่มเป้าหมายที่ได้รับการลงทะเบียน

                     7. สถิติการส่งเคลม สปสช. กรณีบริการการผ่าตัดแบบวันเดียวกลับ (One Day Surgery: ODS) ผ่าน Financial Data Hub
Dashboard แสดงผลการเบิกจ่ายชดเชยค่าใช้จ่ายเพื่อบริการสาธารณสุข กรณีกลุ่มโรค รหัสโรค (ICD-10 2010) และรหัสหัตถการ (ICD-9-CM 2010) ตามเอกสารแนบท้ายหมายเลข 1.2 ประกาศสำนักงานหลักประกันสุขภาพแห่งชาติ เรื่อง การจ่ายค่าใช้จ่ายเพื่อบริการสาธารณสุข กรณีการบริการผ่าตัดแบบ
ไม่ค้างคืนและกรณีการบริการผ่าตัดผ่านกล้อง พ.ศ. 2566 ลงวันที่ 10 มกราคม พ.ศ. 2566 ประกอบด้วย      4 ส่วน ดังนี้ 
                     * การแสดงผลภาพรวมของการเบิกจ่ายชดเชยฯ 
                     * การแสดงผลการเบิกจ่ายชดเชยฯ จำแนกบริการการผ่าตัดแบบวันเดียวกลับ (One Day Surgery: ODS) และบริการผู้ป่วยใน (In-patient Department: IPD) กองทุนหลักประกันสุขภาพแห่งชาติ
                     * ข้อมูลผลให้การบริการของหน่วยบริการ กรณีกลุ่มโรค รหัสโรค (ICD-10 2010) และรหัสหัตถการ (ICD-9-CM 2010) ตามเอกสารแนบท้ายหมายเลข 1.2 ประกาศสำนักงานหลักประกันสุขภาพแห่งชาติ เรื่อง การจ่ายค่าใช้จ่ายเพื่อบริการสาธารณสุข กรณีการบริการผ่าตัดแบบไม่ค้างคืนและกรณีการบริการผ่าตัดผ่านกล้อง พ.ศ. 2566 ลงวันที่ 10 มกราคม พ.ศ. 2566
                     * การวิเคราะห์ข้อมูลผลการบริการผ่าตัดแบบวันเดียวกลับ (One Day Surgery: ODS) กองทุนหลักประกันสุขภาพแห่งชาติ

ส่วนที่ 6 แนวทางการเพิ่มประสิทธิภาพ


แนวทางการให้บริการและการขอรับค่าใช้จ่ายบริการสาธารณสุข ผ่านระบบ Financial Data Hub (FDH) เพื่อเพิ่มประสิทธิภาพการจัดบริการและการบริหารจัดการข้อมูลการเงินการคลังในหน่วยบริการ โดยใช้ระบบดิจิทัลและการบูรณาการข้อมูลร่วมกันอย่างเป็นระบบ ครอบคลุมตั้งแต่การคัดกรองผู้ป่วยจนถึงการเบิกจ่ายค่าบริการผ่านระบบ FDH อย่างถูกต้อง โปร่งใส และสามารถตรวจสอบได้ สามารถดำเนินการได้ 11 ขั้นตอน ดังนี้


  

ภาพที่ 114 แนวทางการเพิ่มประสิทธิภาพ ของหน่วยบริการ 11 ขั้นตอน
ขั้นตอนที่ 1: คัดแยก (Triage)
                        * เจ้าหน้าที่พยาบาลวิชาชีพทำหน้าที่คัดแยกผู้ป่วยเบื้องต้น ตามระดับความเร่งด่วน (Emergency Severity Index - ESI)
                        * ระบุแผนกที่ผู้ป่วยจะเข้ารับการรักษาในลำดับถัดไป เช่น ห้องตรวจเวชกรรม ห้องอุบัติเหตุ ฯลฯ
ขั้นตอนที่ 2: ห้องบัตร (Register)
                        * นักเวชสถิติตรวจสอบข้อมูลผู้ป่วย ตรวจสอบสิทธิการรักษา (UCS/ประกันสังคม/ข้าราชการ/อื่นๆ)
                        * ยืนยันตัวตนด้วยระบบ Authen และบันทึกข้อมูล 16 แฟ้มพื้นฐาน (ตามมาตรฐาน สปสช.)
                        * กรณีเป็นผู้ป่วยส่งต่อ ให้ดำเนินการ Refer ตามระบบ
ขั้นตอนที่ 3: ห้องตรวจ
                        * แพทย์ซักประวัติ ตรวจร่างกาย และบันทึกข้อมูล Vital Signs, BMI
                        * กำหนดรหัสโรค ICD-10, ICD-9 ตามมาตรฐานการวินิจฉัย
                        * ตรวจสอบสิทธิ์การรักษาพยาบาลของคนไข้
                        * ลงชื่อแพทย์ผู้ตรวจและตรวจสอบความครบถ้วนของข้อมูล
ขั้นตอนที่ 4: ข้อมูล Digital Health
                        * เจ้าหน้าที่ด้านคอมพิวเตอร์และสาธารณสุขทำหน้าที่ตรวจสอบความครบถ้วนของ 16 แฟ้มบริการสุขภาพ
                        * Map ประเภทบริการ (ADP Type) และรหัสการรักษา (ADP Code) ให้ตรงกับบัญชีรายโรคและ         ค่ารักษาพยาบาล
                        * เชื่อมโยงข้อมูลจากห้องตรวจ (จุดที่ 3) เพื่อความถูกต้อง
                        * ตรวจสอบความสอดคล้องกับข้อมูล Financial Data Set เพื่อเตรียมปิดสิทธิ์ในขั้นตอนต่อไป
ขั้นตอนที่ 5: การเงิน
                        * จัดทำใบแจ้งหนี้ (Invoice), ปิดสิทธิ์, ตรวจสอบวงเงิน และดำเนินการด้านการเงิน (เงินสด, โอน, รูดบัตร)
                        * ปิดสิทธิ์ผ่านระบบ 6 Minimal Data Set ได้แก่
                        * วันที่ และเวลาที่รับบริการ
                        * เลขบัตรประชาชน 13 หลัก
                        * ข้อมูล HCODE
                        * จำนวนเงิน ที่ขอเบิก (Amount Total)
                        * เลข Visit Number (VN)
                        * เลขใบ Invoice Number
                        * กระบวนการนี้ใช้เฉพาะผู้ที่มีการยืนยันตัวตนแบบ One ID ผ่านระบบ FDH เท่านั้น เพื่อป้องกันข้อมูลซ้ำซ้อน
ขั้นตอนที่ 6: ข้อมูล Digital Health (เพิ่มเติม)
                        * ตรวจสอบสิทธิ์อีกครั้งโดยเจ้าหน้าที่ประกันสุขภาพ
                        * บันทึกข้อมูลผู้มีสิทธิ์ที่เข้าเกณฑ์ได้รับบริการ
                        * จัดประเภทข้อมูลตามรายการบริการ เช่น บริการเฉพาะทาง / P&P / ยา / เวชภัณฑ์ ฯลฯ
                        * เตรียมข้อมูลส่งไปยังระบบตรวจสอบและการเบิกจ่าย
ขั้นตอนที่ 7: ข้อมูล Audit
                        * ตรวจสอบความถูกต้องของข้อมูลผู้ป่วยและข้อมูลบริการ เช่น Authorize Code, Authorization Date
                        * เชื่อมโยงกับระบบประกาศของ สปสช. เพื่อให้เป็นไปตามเงื่อนไขการเบิกจ่าย
ขั้นตอนที่ 8: Pre-Screening
                        * แผนก Digital ทำการคัดกรองข้อมูลล่วงหน้าก่อนเข้าสู่กระบวนการ Claim
                        * ป้องกันความผิดพลาด เช่น ข้อมูลไม่ครบ, รหัสผิด, สิทธิไม่ตรง, จำนวนเงินไม่สัมพันธ์กับประเภทบริการ
ขั้นตอนที่ 9: Claim
                        * จัดทำและรวบรวมยอด SEQ, Claim และยอดจอง Claim โดยแยกประเภทสิทธิ์และรายการบริการที่สามารถเบิกได้
ขั้นตอนที่ 10: สถานะ REP ตอบกลับ
                        * ตรวจสอบผลลัพธ์ของการส่ง Claim ผ่านระบบ REP
                        * วิเคราะห์ผลตอบกลับ เช่น ผ่าน / ไม่ผ่าน / รอข้อมูลเพิ่มเติม
                        * ติดตามสถานะผ่านระบบ Dashboard FDH
ขั้นตอนที่ 11: การติดรหัส C / Deny
                        * ตรวจสอบรายการที่ถูก Reject / ติดรหัส C หรือถูกปฏิเสธ (Deny)
                        * ตรวจสอบความครบถ้วนของเอกสาร STM และขั้นตอนการทบทวนสิทธิ์
                        * ดำเนินการแก้ไขและติดตามผลอย่างต่อเนื่อง

สำหรับเจ้าหน้าที่ไม่มีหมอพร้อมแอปพลิเคชัน ให้ทำการดาวน์โหลดและติดตั้งหมอพร้อมแอปพลิเคชัน หลังติดตั้งแอปพลิเคชันสำเร็จ ให้เลือก สร้างบัญชีใหม่ ตามภาพ หรือลงชื่อเข้าสู่ระบบในกรณีเจ้าหน้าที่มีบัญชี   หมอพร้อมแอปพลิเคชัน


  

2) กรอกเลขประจำตัวบัตรประชาชน และเบอร์โทรศัพท์
กรอกเลขประจำตัวบัตรประชาชน 13 หลัก เบอร์โทรศัพท์ และกดปุ่ม ตรวจสอบข้อมูล 
  

3) ยอมรับข้อกำหนดและเงื่อนไขการใช้บริการ Line Application หมอพร้อม
อ่านและกดปุ่ม ยอมรับ ข้อกำหนดและเงื่อนไขการใช้บริการ Line Application “หมอพร้อม” สำหรับการใช้งานแอปพลิเคชัน ในฐานะบุคคลทั่วไป 
  



3.4 ยินยอมให้หมอพร้อมส่งการแจ้งเตือน
  



   3.5 กรอกข้อมูลส่วนตัวตามแบบฟอร์ม และกดปุ่ม ตรวจสอบข้อมูล 


  











3.6 ตั้งรหัสผ่านเป็นตัวเลข จำนวน 6 หลัก และยืนยันรหัสผ่าน จากนั้นกดปุ่ม ลงทะเบียน
  





2) หน้าเพิ่มเพื่อน LINE Official
         เปิดแอปพลิเคชัน LINE ไปที่เพิ่มเพื่อน ตามหมายเลข 1 และไปที่ Official account เลือก See all ตามหมายเลข 2 จะเข้าสู่หน้า LINE official accounts
